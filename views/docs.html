Properties:

  • tagimacator-data

    Desc: Attribute that resolves to a model on the scope hierarchy to populate tag data.

    Required: true.

    Misc: It is important to note that:

    • Only arrays of objects are currently supported
    • If the attribute resolves to a function it must return a promise.
    • See: here for more info.

    • HTML
    • JS

                
<div tagimacator tagimacator-data="getUsers"><div>
            
          
                
$scope.getUsers = function() {
  var def = $q.defer();

  $http({method:'GET', url:'http://myserver.com/users'}).success(function(users) {
    def.resolve(users);
  });

  return def.promise;
};

  • tagimacator-save-to

    Desc: Attribute that resolves to a model on the scope hierarchy to save selected tag models to.

    Required: true.

    Misc: It is important to note that:

    • Must be an array.
    • See: here for more info.

    • HTML
    • JS

                
<div tagimacator tagimacator-data="getUsers" tagimacator-save-to="usersSave"><div>
            
          
                
$scope.usersSave = [];

$scope.getUsers = function() {
  var def = $q.defer();

  $http({method:'GET', url:'http://myserver.com/users'}).success(function(users) {
    def.resolve(users);
  });

  return def.promise;
};

  • tagimacator-filter-by

    Desc: Attribute that refers to a key name within the objects of the tagimacator-data array that is provided. The filter-by and save-by attributes are strongly related however only filter-by is required. Filter-by concerns how the data set will be queried when typing into the tagimacator. Save-by concerns how the data will be saved when a tag is selected.

    Required: true.


    • HTML
    • JS
    • DATA

                
<div tagimacator tagimacator-data="getUsers" tagimacator-save-to="usersSave" tagimacator-filter-by="name"><div>
            
          
                
$scope.usersSave = [];

$scope.getUsers = function() {
  var def = $q.defer();

  $http({method:'GET', url:'http://myserver.com/users'}).success(function(users) {
    def.resolve(users);
  });

  return def.promise;
};
            
          
                
// Data example
// Filtering by name
{
  age: 42,
  name: "Hank",
  _id: "5362f9d895f21a8c0e3d35cf"
}

  • tagimacator-save-by

    Desc: Attribute that refers to a key name within the objects of the tagimacator-data array that is provided. It is used to save selected tags to the "tagimacator-save-to" model. If not provided it will use the "tagimacator-filter-by" attribute. The reason for the distinction is to prevent duplicates from being selected and saved. The example here has many duplicate "name" fields, but each object has a unique "_id". Thus each "Hank" actually refers to a seperate person.

    Required: false.


    • HTML
    • JS
    • DATA

                
<div tagimacator tagimacator-data="getUsers" tagimacator-save-to="usersSave" tagimacator-filter-by="name" tagimacator-save-by="_id"><div>
            
          
                
$scope.usersSave = [];

$scope.getUsers = function() {
  var def = $q.defer();

  $http({method:'GET', url:'http://myserver.com/users'}).success(function(users) {
    def.resolve(users);
  });

  return def.promise;
};
            
          
                
// Data example
// Filtering by name
// Saving by _id
{
  age: 42,
  name: "Hank",
  _id: "5362f9d895f21a8c0e3d35cf"
}

Methods

  • tagimacator-on-tag-select

    Desc: Attribute that resolves to a function on scope that is used as a callback when a tag is selected.

    Arguments: The only thing passed to the on-tag-select callback is the item model of the tag that was clicked.

    Required: false.


    • HTML
    • JS

                
<div tagimacator tagimacator-data="getUsers" tagimacator-save-to="usersSave" tagimacator-filter-by="name" tagimacator-save-by="_id" tagimacator-on-tag-select="onselect"><div>
            
          
                
$scope.usersSave = [];

$scope.getUsers = function() {
  var def = $q.defer();

  $http({method:'GET', url:'http://myserver.com/users'}).success(function(users) {
    def.resolve(users);
  });

  return def.promise;
};

$scope.onselect = function(itm) {
  console.log(itm);
};
